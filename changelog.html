v1 — Local
The Beginning — Local HTML + Python Proxy
v0.1.0
March 2026, Session 1

What was built

  • Single HTML file served from local Python HTTP server
  • Upstox API proxy over localhost:8080
  • 30 Nifty 50 stocks with ISINs
  • Basic ORB (Opening Range Breakout) signal — first 15 candles
  • VWAP calculated from intraday 1-minute candles
  • RSI(14) from daily closes
  • ATR(14) for stop loss and target calculation
  • BUY / SELL / WATCH signal logic
  • Stock search with dropdown, quick-add pills

v7 — Signals
Market Context Filters (Nifty, Sector, Gap, Day Trend)
v1.2.0
March 2026, Session 2

What was added

  • Nifty 50 hard block: ±0.5% threshold forces counter-trend signals to WATCH
  • Sector filter: Bank Nifty / IT / Auto / Pharma headwind −15%, tailwind +5%
  • Gap filter: gap-down BUY or gap-up SELL reduces confidence −10%
  • Day trend filter: stock net negative on BUY day −10%, positive +5%
  • Market context banner in scanner UI (red/green/amber)
  • Index keys and sector map in both signals.py and JS
  • Context warnings shown inline in detail panel
The HDFCBANK problem that triggered this HDFCBANK showed BUY at ₹812 entry while BSE Sensex was down 2000 points. The scanner was stock-blind — it saw ORB breakout and VWAP above without knowing the entire market was crashing. This filter prevents that mistake.

v8 — Macro
4-Layer Macro Intelligence Engine
v1.3.0
March 2026, Session 2

What was added

  • macro.py — new file, all 4 layers
  • Layer 1: Economic calendar — Investing.com, suppress signals 30 min around high-impact events (RBI, Fed, Budget)
  • Layer 2: Yahoo Finance proxies — Brent crude, Gold, USD/INR, S&P futures, VIX, Dollar Index. Sector-specific penalties applied.
  • Layer 3: NSE FII/DII flows — FII net sellers >₹500 Cr bearish −10%, buyers bullish +5%
  • Layer 4: NewsAPI + Claude Haiku — fetches last 6 hrs headlines, classifies overall and sector-specific sentiment
  • Macro context cached 30 min to stay within NewsAPI free tier (100 req/day)
  • GET /macro-status endpoint — inspect all 4 layers anytime
  • apply_all_macro_penalties() — combines all 4 layers into single penalty
