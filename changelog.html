v1 — Local
The Beginning — Local HTML + Python Proxy
March 2026, Session 1

What was built

  • Single HTML file served from local Python HTTP server
  • Upstox API proxy over localhost:8080
  • 30 Nifty 50 stocks with ISINs
  • Basic ORB (Opening Range Breakout) signal — first 15 candles
  • VWAP calculated from intraday 1-minute candles
  • RSI(14) from daily closes
  • ATR(14) for stop loss and target calculation
  • BUY / SELL / WATCH signal logic
  • Stock search with dropdown, quick-add pills

Files

scanner.html upstox_proxy.py GetUpstoxToken.ps1
Why it was built this way Starting point: a way to view live NSE signals without paying for a trading terminal. Local-first because Upstox CORS policy requires a server-side proxy.

v7 — Signals
Market Context Filters (Nifty, Sector, Gap, Day Trend)
March 2026, Session 2

What was added

  • Nifty 50 hard block: ±0.5% threshold forces counter-trend signals to WATCH
  • Sector filter: Bank Nifty / IT / Auto / Pharma headwind −15%, tailwind +5%
  • Gap filter: gap-down BUY or gap-up SELL reduces confidence −10%
  • Day trend filter: stock net negative on BUY day −10%, positive +5%
  • Market context banner in scanner UI (red/green/amber)
  • Index keys and sector map in both signals.py and JS
  • Context warnings shown inline in detail panel
The HDFCBANK problem that triggered this HDFCBANK showed BUY at ₹812 entry while BSE Sensex was down 2000 points. The scanner was stock-blind — it saw ORB breakout and VWAP above without knowing the entire market was crashing. This filter prevents that mistake.

v8 — Macro
4-Layer Macro Intelligence Engine
March 2026, Session 2

What was added

  • macro.py — new file, all 4 layers
  • Layer 1: Economic calendar — Investing.com, suppress signals 30 min around high-impact events (RBI, Fed, Budget)
  • Layer 2: Yahoo Finance proxies — Brent crude, Gold, USD/INR, S&P futures, VIX, Dollar Index. Sector-specific penalties applied.
  • Layer 3: NSE FII/DII flows — FII net sellers >₹500 Cr bearish −10%, buyers bullish +5%
  • Layer 4: NewsAPI + Claude Haiku — fetches last 6 hrs headlines, classifies overall and sector-specific sentiment
  • Macro context cached 30 min to stay within NewsAPI free tier (100 req/day)
  • GET /macro-status endpoint — inspect all 4 layers anytime
  • apply_all_macro_penalties() — combines all 4 layers into single penalty

v13 — Auth
User Authentication, Sessions & PostgreSQL
April 2026, Session 3

What was added

  • login.html & password-reset.html — dedicated auth UI replacing the inline token card
  • Username/password login with bcrypt hashing (12 rounds)
  • 32-byte random session tokens stored in PostgreSQL (Neon) — 30-day TTL
  • Role-based access: admin (full access) and viewer (read-only)
  • POST /app/login, POST /app/logout, GET /app/me, POST /app/change-password
  • db.py — full database layer: users, app_sessions, token_store, session_state, trade_history, alert_log tables
  • All scanner state (locked signals, macro cache) persisted in PostgreSQL — survives Render restarts
  • History tab: trades now saved to PostgreSQL instead of just localStorage
  • GET /history/trades, POST /history/trades, PATCH /history/trades/<id>, DELETE /history/trades/<id>
  • GET /history/stats — server-side accuracy computation
  • Auth guard on every page: redirects unauthenticated users to login.html
  • X-Session-Token header support for all protected endpoints

v17 — Auto-Login
Morning Auto-Login Reminder (Telegram + Email at 08:30 IST)
May 2026, Session 4

What was added

  • auto_login.py — scheduled job module for the 08:30 IST morning reminder
  • APScheduler cron job: weekdays only, sends Telegram message with one-tap /auth/login link
  • Email login reminder fires simultaneously via email_alerts.format_login_reminder()
  • GET /auth/auto-login-status — shows last reminder attempt, status, and next scheduled run
  • POST /auth/trigger-auto-login — admin can fire the reminder immediately for testing
  • Token expiry detection in scanner: if Upstox returns 401, scanner sets token_expired_alerted flag and fires email
  • Guard on /auth/login: if a valid token already exists, shows "already logged in" page instead of starting a new OAuth (prevents token revocation)
  • Removed requirement to open set-token-form manually every morning — reminder now pushes the link to you

Files

auto_login.py app.py scanner.py
Why Forgetting to login before 9:15 IST means the scanner runs the entire morning without a token — all alerts silently fail. A push reminder at 08:30 eliminates that failure mode. The /auth/login guard prevents the equally painful mistake of logging in twice and revoking your own working token.
