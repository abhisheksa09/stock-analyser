1. Overview & Architecture

The NSE Intraday Scanner is a cloud-hosted trading assistant split across two deployments. The frontend runs on GitHub Pages (free, static). The backend runs on Render.com and handles all live data, signal computation, macro intelligence, OAuth, and Telegram alerts.

File Map

📖
scanner.html
Main SPA on GitHub Pages. Scanner, History, Backtest and Admin tabs. Signal display, market context banner, floating tooltips, AI trade plans.
🔒
login.html
Auth gate. Username/password login. Sessions stored in PostgreSQL. Auth guard on scanner.html, readme.html and changelog.html.
🐍
app.py
Flask web server on Render. 50+ HTTP routes. Upstox + Anthropic proxy. OAuth flow. APScheduler for scan, EOD settlement and morning reminder.
🔎
scanner.py
Background alert engine. Runs every 5 min. Manages session state. Sends Telegram alerts. Auto-saves paper trades on green/amber signals.
📈
signals.py
Signal computation: ORB, VWAP, RSI(14), ATR(14), confidence scoring, 4-filter market context (Nifty, sector, gap, day trend).
🌎
macro.py
4-layer macro intelligence: economic calendar, Yahoo Finance proxies, NSE FII/DII flows, NewsAPI + Claude Haiku sentiment. 30-min cache.
🗄
db.py
PostgreSQL layer (Neon). Tables: users, app_sessions, token_store, session_state, trade_history, alert_log, paper_trades.
💌
email_alerts.py
Resend API integration. 6 email types: green_ready, conf_crossed, reversal, token_expiry, login_reminder, eod_settlement digest.
⏱
auto_login.py
Morning reminder module. Fires Telegram + email tap-to-login link at 08:30 IST on weekdays via APScheduler cron job.
🧠
ai_insights.py
OpenAI gpt-4o-mini integration. Reviews trade setup quality on demand from the stock detail panel in scanner.html.

2. Daily Setup

First-time: App Login

The scanner requires a username and password to access any page. On first visit you will be redirected to login.html. Your admin will create your account via the Admin tab.

🔒 Sessions last 30 daysAfter logging in once, your session is stored in PostgreSQL. You will stay logged in across browser restarts until the session expires or you log out.

Daily Upstox Login (every morning)

The Upstox access token expires at midnight IST. You need to re-authenticate each morning before 9:15 IST. The process is now fully automated:

⏱ 08:30 IST reminder fires automaticallyA Telegram message and email are sent to you at 08:30 IST every weekday with a one-tap login link. No manual action needed until you see the reminder.
1
Receive reminder at 08:30 IST
Telegram and email both arrive with a direct link to /auth/login.
2
Tap the link on your phone
Opens Upstox login page. Enter your mobile number and OTP.
3
Tap Authorize
Render exchanges the code for a token automatically. You see a green success card. The tab closes itself after 3 seconds.
4
Done — scanner is active
Token is saved in PostgreSQL. The status dot in scanner.html turns green. Alerts fire from 9:15–10:30 IST automatically.
⚠ Never tap login twice in the same morningOpening /auth/login when a valid token already exists will show an "already logged in" page and do nothing — this is intentional. Upstox revokes the old token if you start a second OAuth session, which would break your live scanner mid-morning.

If You Need to Re-login Mid-day

If the token expires unexpectedly (Upstox returns 401), you will receive a token expiry email. To fix:

1
Open Admin tab → clear today's token
This removes the stale token so the login guard allows a new OAuth flow.
2
Open /auth/login on phone
Complete Upstox OTP login as normal. New token is saved automatically.
⏳ Render free tier cold startSet up UptimeRobot (free) to ping https://nse-proxy-mojx.onrender.com/ping every 5 minutes so Render never sleeps and never misses market open.

4. Market Context Filters

Applied automatically on every scan. These are the most important additions over a basic technical scanner — they prevent counter-trend trades on strong market days.

Nifty Hard Block (±0.5%)

Nifty changeEffect on BUY signalsEffect on SELL signals
≤ −0.5%🚫 Hard blocked → forced to WATCH✅ Allowed (tailwind)
−0.5% to +0.5%✅ Normal✅ Normal
≥ +0.5%✅ Allowed (tailwind)🚫 Hard blocked → forced to WATCH

When blocked, the signal shows WATCH with reason: "BUY blocked — Nifty −1.6% today. Market too bearish for long trades." A red/green banner appears at the top of the scanner.

Sector Headwind / Tailwind

SituationConfidence impact
BUY signal but sector index < −0.5%−15%
SELL signal but sector index > +0.5%−15%
BUY signal and sector index > +0.5%+5% bonus
SELL signal and sector index < −0.5%+5% bonus

Sector indices used: Bank Nifty (Banking/NBFC), Nifty IT (IT), Nifty Auto (Auto), Nifty Pharma (Pharma). Other sectors use Nifty 50 as fallback.

Gap Filter

If a stock gaps down >0.5% at open and shows a BUY signal, that signal is likely a weak bounce, not a real breakout. Confidence is reduced by 10%. Same applies to gap-up opens on SELL signals.

Day Trend Filter

SituationEffect
BUY + stock net negative on day (≤ −0.5%)−10% confidence
SELL + stock net positive on day (≥ +0.5%)−10% confidence
BUY + stock net positive (≥ +0.5%)+5% bonus
SELL + stock net negative (≤ −0.5%)+5% bonus

5. Macro Intelligence

Four additional data layers run in the background every 30 minutes during market hours. They adjust confidence scores and can suppress signals based on global and domestic macro conditions.

Layer 1 — Economic Calendar Block

Source: Trading Economics / Investing.com

Fetches today's India and global high-impact events. Within 30 minutes of an RBI policy decision, US FOMC, Budget, GDP release, or similar — all signals get −50% confidence penalty, effectively suppressing all trades during the volatile announcement window.

Layer 2 — Macro Proxy Fetchers

Source: Yahoo Finance (no API key needed)

Fetches 6 instruments and applies sector-specific penalties:

InstrumentSymbolWhen it mattersEffect
Brent CrudeBZ=F+2% spike−12% on Auto/FMCG/Pharma BUYs. +8% on Energy BUYs.
GoldGC=F+1.5% rallyRisk-off signal — −10% on all equity BUYs
USD/INRUSDINR=XINR weakens >0.5%−10% Auto/FMCG/Pharma BUYs. +8% IT BUYs (export earnings).
S&P 500 futuresES=F< −1.5%Global risk-off — −15% on all BUYs
VIX^VIX>25−10% all signals. >30 = −20% (extreme fear)
Dollar IndexDX-Y.NYBDXY >+0.5%FII outflows likely — indirect bearish signal

Layer 3 — FII/DII Flow Data

Source: NSE India public API (no key needed)

NSE publishes daily institutional buying/selling. FII net sellers >₹500 Cr — bearish bias, −10% on BUY signals. FII net buyers — bullish, +5% on BUY signals. DII flows used as secondary signal.

Layer 4 — Claude News Sentiment

Source: NewsAPI.org headlines + Claude Haiku classification

Fetches last 6 hours of India market headlines. Claude Haiku classifies them into overall market bias (bullish/bearish/neutral) and sector-specific impacts. Refreshes every 30 minutes to stay within NewsAPI free tier (100 req/day). Confidence confidence: high=full weight, medium=60%, low=30%.

News scenarioEffect on BUYEffect on SELL
Overall bearish (high confidence)−15%+5% bonus
Overall bullish (high confidence)+5% bonus−15%
Sector-specific bearish news−10%+5%

7. Signal Lock & Reversal Detection

The scanner remembers every signal for each stock across all scans in the same session.

  • Session lock — first signal of the day is locked (🔒). Shown below the stock name on every subsequent scan.
  • Signal history dots — B/S/W circles (oldest → newest). Current scan has a black ring.
  • Reversal badge — if current signal differs from locked signal, a red ⚡ REVERSED badge appears and the row gets a red left border.
  • Reversal banner — red banner at top of scanner listing all reversed stocks.
  • Session reset — midnight IST automatically resets all locks and history.
⚡ Reversed signal = skipA stock that flips direction within minutes is ranging, not trending. The Telegram scanner also suppresses alerts for reversed stocks.

8. Candle Confirmation

The last 3 consecutive one-minute candles must all close on the correct side of VWAP before a signal is confirmed. Unconfirmed signals get a −20% confidence penalty automatically.

9. Confidence Engine

Base confidence is computed from 6 technical factors, then adjusted by market context filters and macro penalties.

FactorWeightScores 100% when…
ORB breakout25%Price confirmed above ORB High (BUY) or below ORB Low (SELL)
Volume confirmation20%Today’s volume ≥1.5× 20-day average
VWAP alignment20%Price on correct side of VWAP for signal direction
RSI alignment15%RSI in 40–60 neutral zone
Risk:Reward15%R:R ≥3:1
ATR / volatility5%Daily range 0.8–2.5% of price

Adjustments applied on top of base score:

  • WATCH signals capped at 45%
  • Unconfirmed candles: −20%
  • Nifty/sector headwind: −15%
  • Gap-down BUY or gap-up SELL: −10%
  • Day trend against signal: −10%
  • S&P futures red: −15%
  • VIX >25: −10% to −20%
  • FII net sellers: −10%
  • News bearish (high confidence): −15%
  • Economic calendar event window: −50%

10. Technical Indicators

ORB — Opening Range Breakout

First 15 one-minute candles (9:15–9:30 AM IST). Highest high = ORB High. Lowest low = ORB Low. BUY requires close above ORB High. SELL requires close below ORB Low.

VWAP — Volume Weighted Average Price

Calculated from all intraday 1-minute candles, resets at 9:15 AM IST daily. Price above VWAP = buyers in control. Also used for candle confirmation checks.

RSI (14)

Calculated from the last 14 daily closes. >70 = overbought, <30 = oversold, 40–60 = neutral (ideal for entries).

ATR (14)

Average True Range from 14 daily candles. Sets stop loss width (0.3–0.4×ATR) and target distance (1.5–2.0×ATR). Ideal range: 0.8–2.5% of price.

14. Trading Best Practices

ConfidenceVerdictMax positionMax risk/trade
75%+🌺 ReadyFull size1% of capital
55–74%🉱 Half sizeHalf size0.5% of capital
<55%🔴 Not readyNo trade—

Hard rules

  • Never enter before 9:45 AM IST
  • Never trade a reversed signal (⚡)
  • Never trade when market is hard-blocked (Nifty ±0.5%)
  • Never trade during economic calendar event window
  • Never risk more than 1% of total capital per trade
  • Max 3 open positions at any time
  • Stop trading for the day after 2 consecutive losses
  • Place stop loss order immediately after entry
  • Use limit orders at Entry price, never market orders
  • Exit all positions by 3:15 PM
  • Never trade on corporate event ex-date

17. Backtest Tab

The Backtest tab shows auto-saved paper trades from the background scanner and their EOD-settled outcomes. No manual input required — the system records and closes trades automatically.

How Paper Trades Are Saved

Every time the background scanner fires a green or amber alert, it automatically saves a paper trade to PostgreSQL if the stock's confidence meets the threshold (BACKTEST_MIN_CONF, default 55%). The saved fields are: symbol, sector, signal, confidence, signal price, entry, target, stop loss, R:R, RSI, and reason.

Env varDefaultEffect
BACKTEST_SYMBOLS12 default stocks across 8 sectorsComma-separated list to override which stocks get paper-traded
BACKTEST_MIN_CONF55Minimum confidence % for a paper trade to be saved
📊 One paper trade per stock per dayThe scanner deduplicates — once a paper trade is saved for a symbol on a given date, no further saves happen for that symbol that day.

EOD Settlement

At 15:35 IST every weekday an APScheduler job replays intraday 1-minute candles from the signal time forward to find the first candle where price hits the target or stop loss.

OutcomeCondition
WonA candle's high (BUY) or low (SELL) touches the target before SL is hit
LostA candle's low (BUY) or high (SELL) touches the stop loss first
Partial WinNeither level hit by session end — exit at intraday best price, positive P&L
Partial LossNeither level hit by session end — exit at intraday best price, negative P&L

When both levels are touched in the same candle, the stop loss is assumed to have been hit first (conservative). An EOD email digest is sent after settlement listing all trades and outcomes.

Walk-Forward Validation

GET /paper-trades/walk-forward?months=6 returns a per-month breakdown of settled paper trades: trade count, win rate, avg P&L, and total P&L for each calendar month. Consistent win rate across months indicates the strategy is robust rather than lucky.
